## 컴퓨터의 네 가지 핵심 부품
### 메모리 (RAM)
- 현재 **실행되는 프로그램의 명령어와 데이터를 저장**하는 부품.
- **프로그램이 실행되려면 반드시 메모리에 저장**되어 있어야 한다.
> ‼️ ROM - 메모리의 일종으로 부팅 프로그램(BIOS)이나 하드웨어 초기설정 등을 담는 용도로 사용, **비휘발성** 메모리.

### 중앙처리장치 (CPU)
- 메모리에 저장된 명령어를 읽기, 해석, 실행하는 부품.
- **산술논리장치(ALU), 레지스터(Register), 제어장치(Control Unit)** 등으로 구성.  
    #### 산술논리장치(ALU)  
    - 컴퓨터 내부에서 수행되는 대부분의 **계산**을 수행
       |계산 종류|수행 연산|
       |--|--|
       |산술 연산|ADD SUB INC DEC MUL DIV|
       |논리 연산|AND OR XOR NOT|
       |비트 조작|Shift Rotate|
       |비교 연산|CMP → 플래그(ZF, CF, SF, OF) 설정  
    👉🏻 [연산에 대해 자세히](#alu-op)

    #### 레지스터(Register)
    - CPU 내부에 위치한 작은 **임시 저장 장치**.
    - CPU 내부에 여러 개의 각기 다른 역할과 이름의 레지스터가 존재함.

    #### 제어장치(Control Unit)
    - **제어신호**를 내보내고 **명령어를 해석**하는 장치.

### 보조기억장치
- 데이터를 보관할 목적의 **저장장치**
- 메모리보다 **크기가 크고** 전원이 꺼져도 저장된 내용을 잃지 않는 **비휘발성**이다.

### 입출력장치
- **컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 교환**하는 장치.

### + 메인보드
- 컴퓨터를 구성하는 **핵심 부품들을 연결**하는 부품.
- 연결된 부품들은 메인보드 내부 **시스템 버스(System Bus)** 를 통해 정보를 주고받는다.  
    |버스 종류|역할|
    |--|--|
    |주소 버스|주소를 주고받는 통로|
    |데이터 버스|명령어와 데이터를 주고받는 통로|
    |제어 버스|제어 신호를 주고받는 통로|

## 보충 이론

### <span id="alu-op">ALU 연산</span>
#### 산술 연산(Arithmetic Operations)  
|종류|기능|종류|기능|
|--|--|--|--|
|ADD|덧셈|MUL|곱셈|
|SUB|뺄셈|DIV|나눗셈|
|INC|증가|NEG|부호 반전|
|DEC|감소|

#### 논리 연산(Logic Operations)
|종류|기능|
|--|--|
|AND|둘다 1일 때만 1|
|OR|둘중 하나라도 1이면 1|
|XOR|서로 다르면 1 같으면 0|
|NOT|비트 반전 0 → 1 / 1 → 0|

#### 비트 조작(Bitwise Operations)
|종류|기능|효과|
|--|--|--|
|Shift Left(SHL)|0001 << 1 → 0010|x2|
|Shift Right(SHR)|0010 >> 1 → 0001|÷2|
|Rotate Left(ROL)|1011 ROL 1 → 0111|x2|
|Rotate Right(ROR)|1011 ROR 1 → 1101|÷2|
> ‼️ Rotate는 밀려난 비트가 반대쪽으로 돌아옴

#### 비교 연산(Comparison)
|종류(Flag)|조건|용도|
|--|--|--|
|ZF|A = B = 0|같음을 판별|
|SF|A - B = -C|부호 있는 정수에서 크다 작다 판별|
|CF|unsigned 수행|unsigned 비교용|
|OF|오버플로우 발생|signed 연산에서 범위를 벗어남을 판별|
> ‼️ 연산 자체가 비트로 진행되기 때문에 플래그 라는 공간에 0과 1을 사용하여 비교를 수행한다.
